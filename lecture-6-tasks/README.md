# Personal Portfolio — Abdul Manan Khalid

A single-page personal portfolio built for the Panaversity **AI-101 · Lecture 6** task ("Markdown In, HTML Out"). It positions me as an **Agentic AI Engineer** with a Mechatronics background, and is written as one self-contained `index.html` file — no external CSS, JavaScript, or fonts, with the profile photo embedded directly in the file.

**Live site:** https://iamkhalid.netlify.app/

**Source:** https://github.com/manan491/ai-101-tasks/tree/main/lecture-6-tasks

## What's inside

- **About, Skills, Projects, Contact** sections plus a name heading
- Dark / light theme toggle
- A sticky status bar with scroll-spy navigation (the current section's nav link glows as you scroll)
- Per-project technology tags
- Mobile-friendly responsive layout
- SEO and Open Graph meta tags
- Fully self-contained: one HTML file, no build step, works offline

## Tech

Plain HTML, CSS, and vanilla JavaScript. No frameworks, no dependencies. The layout uses CSS Grid and Flexbox; theming is done with CSS custom properties; navigation and theme switching use a small amount of vanilla JS.

## How it was built

Built through the Markdown → HTML AI-assisted workflow taught in class: I wrote myself in Markdown first, then iterated with an AI tool across several rounds to generate and refine the HTML, grade the design, and fix issues.

### Prompts used

1. "Turn this into a clean, mobile-friendly, one-page portfolio website in HTML. Use clear sections, a nice colour theme, and make it look professional. Give me a single HTML file."
2. "Grade this page out of 10 for design and readability, then improve it."
3. "Make it fully self-contained — remove external font dependencies."
4. "Add a dark/light theme toggle, a sticky nav, and per-project tech tags."
5. "Add scroll-spy so the nav link for the current section glows as I scroll."
6. "Embed my photo as the profile picture, kept inside the single file."
7. "Review the whole file, find UI bugs, and fix them."

## Running it

Open `index.html` in any browser — that's it. To publish, enable GitHub Pages on this repo (Settings → Pages → deploy from `main`), or drag the file onto Netlify.
